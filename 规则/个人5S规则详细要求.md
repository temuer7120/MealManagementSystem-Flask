
# 敏捷开发5S个人规则.md

敏捷开发5S个人规则，是将敏捷5S规则落地到个人层面，核心是“规范个人行为、提升个人效率、适配团队敏捷节奏”，让每一位团队成员都能通过践行5S规则，实现“个人高效、团队协同”，避免因个人行为混乱导致团队迭代受阻，贴合个人实操场景，简单易落地、可执行性强。

## 一、个人5S规则核心原则

以“价值为核心、效率为目标、规范为基础、持续为导向”，不追求形式化，聚焦“个人能落地、能坚持、能产生价值”，贴合敏捷开发“快速响应、持续迭代”的节奏，兼顾个人工作习惯与团队协同需求。

## 二、个人5S规则详细要求

### 1. Sort（个人整理）—— 分清轻重缓急，拒绝无效内耗

核心：个人层面，筛选出对“迭代目标、个人成长”有价值的工作，剔除无效事务，避免精力浪费，确保每一份精力都用在核心事情上。

个人实操：

* 每日上班前10分钟，梳理当日任务，按“紧急重要、紧急不重要、重要不紧急、不紧急不重要”分类，优先完成紧急重要任务；
* 拒绝“无效忙碌”：不参与无关的沟通、不处理与当前迭代无关的事务，遇到非本职、非紧急的事情，及时反馈、合理推诿；
* 定期清理个人工作空间（线上+线下）：删除无用文件、清理桌面杂物，卸载不常用的工具，避免环境杂乱影响专注力。

### 2. Set in Order（个人整顿）—— 有序规划，快速高效

核心：将个人的任务、文档、工具进行有序规划，形成固定的存放与执行习惯，确保“需要时能快速找到、执行时能高效推进”，减少时间浪费在“找东西、理思路”上。

个人实操：

* 任务整顿：使用个人任务管理工具（如备忘录、飞书任务），记录每日/每周任务，标注截止时间、优先级，实时更新任务状态（未开始、进行中、已完成）；
* 文档整顿：个人文档按“项目名称、迭代版本、需求类别”分类存放，命名规范（如“XX项目-202602迭代-个人开发笔记”），重要文档及时备份、同步共享；
* 工具整顿：固定个人常用工具的摆放/打开位置，熟练掌握核心工具（如开发工具、测试工具、沟通工具）的使用技巧，减少工具切换与操作耗时。

### 3. Shine（个人清扫）—— 及时清零，保持高效状态

核心：个人层面，及时清理工作中的“问题、冗余、拖延”，保持高效的工作状态，不堆积问题、不拖延任务，确保个人工作不影响团队迭代进度。

个人实操：

* 问题清扫：开发/工作中遇到的问题（如Bug、需求疑问），及时记录、主动沟通解决，不堆积、不回避；当日问题尽量当日清零，无法解决的及时上报，同步进度；
* 任务清扫：每日下班前，清理当日未完成的任务，分析未完成原因，调整次日计划；迭代过程中，不拖延核心任务，避免因个人拖延导致团队迭代延期；
* 状态清扫：工作中保持专注，避免分心（如刷手机、无关聊天），遇到情绪内耗、工作疲惫时，适当调整，快速恢复高效状态，不影响团队协同节奏。

### 4. Standardize（个人标准化）—— 建立习惯，规范行为

核心：将“整理、整顿、清扫”的个人行为，固化为个人工作习惯，形成标准化的个人工作模式，确保每一项工作都有章可循、不混乱。

个人实操：

* 流程标准化：固定个人工作流程（如开发流程：需求理解→方案设计→编码→自测→提交），每一步都按规范执行，避免遗漏关键环节；
* 交付标准化：个人交付的工作成果（如代码、笔记、测试用例），按团队规范执行，确保格式统一、内容完整，减少返工；
* 沟通标准化：主动同步个人工作进度（如每日站会精准发言、遇到问题及时同步），沟通时简洁明了，不啰嗦、不模糊，提升沟通效率。

### 5. Sustain（个人素养）—— 自觉践行，持续提升

核心：将个人5S规则内化为自觉行为，不需要他人监督，就能主动践行，同时持续反思、持续优化，实现个人效率与能力的双重提升，适配团队敏捷成长节奏。

个人实操：

* 自觉践行：主动遵守个人5S规则，不敷衍、不松懈，养成“整理、整顿、清扫”的习惯，让规范成为本能；
* 持续反思：每日/每周反思个人5S执行情况，分析自身存在的问题（如拖延、文档杂乱），制定改进措施，及时优化；
* 协同提升：践行个人5S规则的同时，配合团队的5S执行，主动提醒身边同事，带动团队共同进步；同时结合团队迭代需求，持续提升个人能力，适配团队敏捷节奏。

## 三、个人5S规则践行小贴士

* 从小事做起：不需要一次性做到完美，先从“每日整理任务、清理文档”等小事入手，逐步养成习惯；
* 量力而行：结合个人工作习惯，调整5S规则的实操方式，确保可坚持，不盲目追求“标准化”而忽略实用性；
* 聚焦价值：所有个人5S行为，都围绕“提升效率、交付价值、配合团队”展开，不做形式化的无用功；
* 持续迭代：个人5S规则不是固定不变的，可结合团队需求、个人成长，持续优化调整，让规则更贴合自身、更具价值。

## 四、个人5S与团队敏捷的关联

团队敏捷的高效运转，离不开每一位成员的自觉践行；个人5S规则的落地，是团队5S规则落地的基础，更是敏捷迭代高效推进的保障：

* 个人高效，才能实现团队高效：每一位成员践行5S规则，提升个人效率，才能避免因个人拖延、混乱导致团队迭代受阻；
* 个人规范，才能实现团队协同：个人按规范执行工作、交付成果、沟通协作，才能减少团队内耗，实现跨岗高效联动；
* 个人成长，才能推动团队进阶：个人持续践行5S规则、持续提升能力，才能带动团队整体素养提升，适配敏捷开发的快速变化。
