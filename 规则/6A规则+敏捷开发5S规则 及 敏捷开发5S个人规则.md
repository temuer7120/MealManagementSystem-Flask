# 6A规则+敏捷开发5S规则.md

## 一、6A规则（敏捷适配版）

6A规则核心是通过“明确目标、聚焦价值、快速响应、持续优化”，为敏捷开发提供底层行为准则，确保团队协作高效、交付精准，适配敏捷迭代、快速试错的核心需求，每一条规则均贴合敏捷开发的实操场景，避免形式化，聚焦落地性。

### 1. A1：Align（对齐）—— 目标同频，上下同心

核心：所有团队成员、所有迭代任务，均对齐产品核心价值与项目整体目标，杜绝“各自为战”。

实操要求：

* 迭代启动前，明确本次迭代的核心目标（OKR），确保人人清晰“做什么、为什么做”；
* 每日站会同步进度，及时校准偏差，避免任务方向与整体目标脱节；
* 跨角色（产品、开发、测试）对齐需求认知，减少沟通壁垒，确保对需求的理解一致。

### 2. A2：Agile（敏捷）—— 灵活应变，快速响应

核心：摒弃“一刀切”的固定流程，以“快速响应变化”为核心，兼顾效率与质量。

实操要求：

* 接受需求的合理变更，建立简单高效的变更反馈机制，不抗拒变化、不拖延响应；
* 迭代周期不宜过长（建议1-2周），确保快速交付可用版本，及时收集反馈；
* 避免过度规划，聚焦“当下可落地”，预留10%-20%的缓冲时间，应对突发问题。

### 3. A3：Accurate（精准）—— 精准交付，减少内耗

核心：每一项任务、每一次交付，都聚焦“价值产出”，杜绝无效工作与重复劳动。

实操要求：

* 需求拆解精准，拆分为可落地、可量化、可测试的小任务（Story点），避免模糊化需求；
* 开发过程中聚焦核心功能，不做“锦上添花”的无效优化，优先保证核心价值交付；
* 测试环节精准定位问题，不遗漏关键Bug，不浪费时间在无效测试用例上。

### 4. A4：Collaborate（协作）—— 跨岗协同，高效联动（注：此处6A中“Collaborate”以“A”类核心逻辑归类，贴合敏捷协同需求）

核心：打破角色壁垒，实现“人人参与、人人负责”的协同模式，提升团队整体效率。

实操要求：

* 每日站会控制在15分钟内，同步“昨日完成、今日计划、遇到的阻碍”，及时协同解决问题；
* 跨角色结对工作（如开发与测试结对），提前沟通需求细节与测试要点，减少返工；
* 建立共享文档，同步迭代进度、需求变更、问题解决方案，确保信息互通无壁垒。

### 5. A5：Assess（复盘）—— 持续复盘，迭代优化

核心：每一次迭代结束后，不回避问题、不敷衍总结，通过复盘实现“迭代一次、进步一次”。

实操要求：

* 迭代复盘会聚焦“亮点、问题、改进措施”，不纠结于“是谁的问题”，只关注“如何改进”；
* 对复盘出的问题，明确责任人、改进时限，确保落地执行，不流于形式；
* 将改进措施融入下一次迭代计划，形成“交付-复盘-优化-再交付”的闭环。

### 6. A6：Advance（进阶）—— 持续学习，共同成长

核心：团队与个人同步进阶，适配敏捷开发的快速变化，提升核心竞争力。

实操要求：

* 定期组织技术分享、敏捷方法学习，同步行业最佳实践，弥补团队能力短板；
* 鼓励团队成员尝试新工具、新方法，提升工作效率，不固守传统模式；
* 关注个人成长需求，结合团队目标，为成员提供成长空间，实现“个人与团队共进阶”。

## 二、敏捷开发5S规则

敏捷开发5S规则，核心是“简化流程、聚焦价值、规范行为、提升效率”，适配敏捷迭代的快节奏，同时避免流程混乱，确保每一项工作都有章法、有价值，本质是“用简单的规则，解决复杂的协作问题”，贴合敏捷“轻流程、重落地”的核心理念。

### 1. Sort（整理）—— 分清必要与不必要，杜绝无效工作

核心：筛选出对“价值交付”有帮助的任务、流程、工具，剔除无效冗余，聚焦核心。

实操要求：

* 迭代需求整理：优先保留核心功能需求，非核心需求（锦上添花类）放入需求池，不纳入当前迭代；
* 工作内容整理：每日梳理自己的任务，剔除“无效沟通、重复劳动”，聚焦当日核心工作；
* 工具与文档整理：清理无用的文档、冗余的工具，保留核心工具与共享文档，避免信息杂乱。

### 2. Set in Order（整顿）—— 有序规范，快速取用

核心：将必要的工作内容、工具、文档，进行有序分类、规范存放，确保“需要时能快速找到、快速使用”，减少时间浪费。

实操要求：

* 任务整顿：通过看板（Kanban）将任务分为“待处理、进行中、已完成、待测试”，状态清晰可见；
* 文档整顿：共享文档按“迭代版本、需求类别、问题记录”分类存放，命名规范（如“202602迭代-核心需求文档”）；
* 工具整顿：团队统一使用核心工具（如项目管理工具、沟通工具），避免多工具切换导致的效率损耗。

### 3. Shine（清扫）—— 及时清理，保持高效环境

核心：不仅是物理环境的清扫，更是“工作环境、流程环境”的清扫，及时清理问题、冗余，保持团队高效运转。

实操要求：

* 问题清扫：开发过程中遇到的Bug、沟通中的分歧，及时解决，不堆积；迭代结束后，清理未完成的冗余任务，更新需求池；
* 流程清扫：定期梳理敏捷流程（如站会、复盘会），剔除无效环节（如冗长的汇报），确保流程高效；
* 环境清扫：保持办公环境（物理/线上）整洁，线上环境不堆积无用文件，物理环境整洁有序，提升专注力。

### 4. Standardize（标准化）—— 建立规范，形成习惯

核心：将“整理、整顿、清扫”的成果固化为标准，形成团队共识，避免流程混乱，确保每一次迭代都能稳定高效。

实操要求：

* 流程标准化：明确站会、复盘会、需求评审会的时间、流程、参与人员，形成固定规范；
* 交付标准化：明确迭代交付物（如代码、测试报告、用户手册）的规范，确保交付物统一、可复用；
* 行为标准化：建立团队共同的行为准则（如及时响应消息、不拖延任务、主动协同），形成良好工作习惯。

### 5. Sustain（素养）—— 坚守规范，持续践行

核心：将5S规则内化为团队成员的自觉行为，不需要强制约束，就能主动践行，形成“人人守规范、事事讲效率”的团队氛围，这是敏捷5S规则的核心目标。

实操要求：

* 团队成员主动践行前4S规则，互相监督、互相提醒，不敷衍、不抵触；
* 将5S规则纳入团队复盘，定期检查执行情况，及时优化调整，避免规则流于形式；
* 培养“主人翁”意识，每个人都主动维护团队的高效环境，推动5S规则持续落地。

### 三、6A规则与敏捷5S规则的关联

6A规则聚焦“目标、协同、优化”，为敏捷开发提供“顶层逻辑”；敏捷5S规则聚焦“流程、规范、行为”，为敏捷开发提供“实操方法”，两者相辅相成：

* 6A的“Align（对齐）”对应5S的“Sort（整理）”：均强调“聚焦核心，剔除冗余”，确保目标与工作同频；
* 6A的“Agile（敏捷）”“Accurate（精准）”对应5S的“Set in Order（整顿）”“Standardize（标准化）”：均强调“灵活且规范，高效且精准”；
* 6A的“Assess（复盘）”“Advance（进阶）”对应5S的“Shine（清扫）”“Sustain（素养）”：均强调“持续优化，形成闭环”，推动团队持续成长。
